# ADRQTCC — the garden of decisions

A long name, kept long on purpose: it holds the whole life of a decision, from
the first murmur to the settled record.

## The vocabulary

- **QTCC** — the four things one can raise: a **Q**uestion, a **T**hought, a
  **C**omment, or a **C**oncern. "QTCCs?" is an invitation to raise yours.
- **ADR** — an *Architecture Decision Record*: a QTCC that has been answered and
  settled — the decision, and the reasoning that earned it, kept where it can be
  found again.
- **bug** — an intent/behavior gap *at any altitude*: the distance between what
  was meant and what happens, whether in a line, a system, or the whole design.

## How an entry grows

A QTCC is raised and lives as an open question. When it's decided it becomes an
ADR — the same file, a new state — carrying its reasoning so the *why* outlives
the moment it was clear. Nothing is deleted: a reversed decision is a new entry
that names the one it replaces. This is a garden, not a whiteboard. What's
planted here is meant to keep.

## Adding one

Copy `TEMPLATE.md` to `NNNN-short-slug.md`, the next number up. Fill what you
know and leave the rest; an entry may begin as a QTCC and earn its way to an ADR.
