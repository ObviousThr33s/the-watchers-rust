# The `.being` safety contract

What no content file can break. Every `.being` is authored by anyone, so
the engine promises these guarantees hold for *any* bytes you put in a file.
Each line below is one such promise, taken straight from a test.

> Generated by `scripts\tests-content.ps1`. Do not edit by hand -- edit the tests.

- An oversized numeric field is a typed error not an overflow.
- An out of range unsigned field is rejected not truncated.
- A multibyte glyph is read whole not sliced.
- A very large file does not panic.
- A storm of separators does not panic.
- Crlf line endings parse without carrying the carriage return.
- A whitespace only header line is skipped not an error.
- Control bytes in an art block are preserved not panicked.
- A key with no value is a typed error.
- Empty input yields a default being.
