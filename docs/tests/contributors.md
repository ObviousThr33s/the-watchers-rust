# Test catalog -- the guarantees

The test suite, read as plain sentences. Each test is one promise the engine
keeps; a red one means a promise was lost. Grouped by file.

> Generated by `scripts\tests-contributors.ps1`. Do not edit by hand -- edit the tests.

## safety_being.rs

_Absolute-safety tests for the `.being` parser — its hostile-input contract._

- An oversized numeric field is a typed error not an overflow.
- An out of range unsigned field is rejected not truncated.
- A multibyte glyph is read whole not sliced.
- A very large file does not panic.
- A storm of separators does not panic.
- Crlf line endings parse without carrying the carriage return.
- A whitespace only header line is skipped not an error.
- Control bytes in an art block are preserved not panicked.
- A key with no value is a typed error.
- Empty input yields a default being.

## safety_haps.rs

_Absolute-safety tests for the `Haps` event ring — the bounded-bus contract._

- Events are copy pure data.
- A zero capacity ring never panics.
- Popping an empty ring yields none forever.
- A full ring rejects overflow without growing.
- Interleaved push pop stays bounded and fifo.
