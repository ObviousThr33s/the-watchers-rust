## safety_being.rs

### `an_oversized_numeric_field_is_a_typed_error_not_an_overflow`
**Says:** An oversized numeric field is a typed error not an overflow.

**Rationale:** A number too large for its field comes back as a typed error, not a silent wraparound — `i32::from_str` rejects the overflow before it can wrap.

### `an_out_of_range_unsigned_field_is_rejected_not_truncated`
**Says:** An out of range unsigned field is rejected not truncated.

**Rationale:** `bloom` is a `u16`; a negative or out-of-range value is rejected, never truncated into a different number.

### `a_multibyte_glyph_is_read_whole_not_sliced`
**Says:** A multibyte glyph is read whole not sliced.

**Rationale:** A glyph is taken as one *character*, not one byte — a multibyte scalar must not be sliced mid-codepoint (which would panic).

### `a_very_large_file_does_not_panic`
**Says:** A very large file does not panic.

**Rationale:** Pathologically large input parses without panicking or stack-blowing.

### `a_storm_of_separators_does_not_panic`
**Says:** A storm of separators does not panic.

**Rationale:** Thousands of block separators are arrival data, not an error or a crash.

### `crlf_line_endings_parse_without_carrying_the_carriage_return`
**Says:** Crlf line endings parse without carrying the carriage return.

**Rationale:** Windows line endings (`\r\n`) parse cleanly — the trailing `\r` is trimmed, not carried into a value or a key.

### `a_whitespace_only_header_line_is_skipped_not_an_error`
**Says:** A whitespace only header line is skipped not an error.

**Rationale:** A header line that is only whitespace is skipped, not treated as a malformed `key value` pair.

### `control_bytes_in_an_art_block_are_preserved_not_panicked`
**Says:** Control bytes in an art block are preserved not panicked.

**Rationale:** Control bytes (including NUL) inside an art block are preserved verbatim, not a parse failure or a panic — the art is kept exactly.

### `a_key_with_no_value_is_a_typed_error`
**Says:** A key with no value is a typed error.

**Rationale:** A header key with no value is a typed error, never an index panic.

### `empty_input_yields_a_default_being`
**Says:** Empty input yields a default being.

**Rationale:** Empty input is the default being, not a crash — the floor case is safe.

## safety_haps.rs

### `events_are_copy_pure_data`
**Says:** Events are copy pure data.

**Rationale:** Events are pure, copyable data (engine ward 1). This is a compile-time proof: if `Event` ever grows a reference, lifetime, or heap field, it stops being `Copy` and this crate fails to build.

### `a_zero_capacity_ring_never_panics`
**Says:** A zero capacity ring never panics.

**Rationale:** A zero-capacity ring is degenerate but must still be panic-free: it accepts nothing and yields nothing, and never divides by its capacity.

### `popping_an_empty_ring_yields_none_forever`
**Says:** Popping an empty ring yields none forever.

**Rationale:** Popping an empty ring yields `None` indefinitely — no underflow, no panic.

### `a_full_ring_rejects_overflow_without_growing`
**Says:** A full ring rejects overflow without growing.

**Rationale:** Once full, every further push is rejected and the ring does not grow — the bounded-allocation ward holds under sustained overflow.

### `interleaved_push_pop_stays_bounded_and_fifo`
**Says:** Interleaved push pop stays bounded and fifo.

**Rationale:** The core invariant under a long, deterministic mix of pushes and pops: `len` never exceeds `CAP`, and the ring always agrees with a simple FIFO model — nothing is lost, duplicated, or reordered. Deterministic (a seeded xorshift) so a failure reproduces exactly.
